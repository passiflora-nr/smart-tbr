# SmartTBR - MVP ideas

### Main problem

The reader has their "To Be Read" (TBR) book lists scattered across multiple inconsistent places (saved Instagram posts, Amazon bookmarks, and loose notes on their phone) and with a backlog of over 100 books, the user wastes time trying to find a title that matches their current mood and literary preferences.

### Smallest set of functionalities

- A basic user account system to store the TBR for the user
- Manual creation of book entries (title, author, description, length, tropes etc.)
- Browsing, editing and deleting book entries
- A simple selection screen where the user chooses which tropes match their current reading mood
- Simple book recommendation flow based on selected literary tropes (e.g. enemies to lovers, grumpy-sunshine), which suggests 5 matched books along with descriptions from the user's TBR backlog

### Not included in MVP

- AI-generated book details or summaries
- Book recommendation flow using AI
- Importing book entries
- Integration with other platforms
- Sharing the TBR with other users
- Mobile application (in the beginning only web app)
- Complex recommendation algorithms (only basic relational database query that checks for matching tags)

### Success criteria

- At least 75% of users agree that the suggested books accurately match the specific tropes they selected
- At least 75% of users state that they no longer need to keep active reading lists elsewhere because they have migrated to SmartTBR
